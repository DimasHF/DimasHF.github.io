Accounting Software

This is a fairly comprehensive accounting web application built for internal use by a business to manage and monitor their financial activities. It covers key features like balance sheets, profit and loss reports, general ledger details, summaries, and categorized transactions. The goal was to make a simple UI but powerful enough for real accounting needs. I was responsible for building the backend system and designing the APIs that handled all the financial logic.

What I did:

  • Built the entire backend using Laravel and designed the database schema for transactions, categories, and user roles.
  • Created RESTful APIs to handle income, expense, balance tracking, and reporting.
  • Implemented monthly and custom-date range report.

Challenges & How I Solved Them:

  • Some features were too finance-specific and unclear at the start.
  • Solution: I regularly communicated with the finance team to understand how they do things manually and turned that into usable system flows.
  • Handling report generation for large data ranges.
  • Solution: I solved this by building a separate microservice using Go, specifically for generating reports like balance sheets and profit & loss. I also used raw SQL queries to speed up data processing and reduce the load compared to using an ORM.
