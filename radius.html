RADIUS Server

This project was built for a local ISP to manage their customer network using a RADIUS server (FreeRADIUS). It handles everything from user authentication, traffic monitoring, to manual billing. It also includes features like simple accounting and a support ticket system so customers can report connection issues easily.

What I did:

  • Built the backend from scratch, including database design.
  • Integrated FreeRADIUS with a dynamic user system via PostgreSQL.
  • Created APIs for traffic reports, billing, ticket system, and session logs.
  • Made sure admins could manage customer payments and troubleshoot easily.
  • Optimized sync between RADIUS logs and app DB using background jobs.

Challenges & How I Solved Them:

  • Real-time RADIUS logs were too heavy.
  • Solution: I used raw SQL instead of relying on Laravel’s Eloquent ORM for performance optimization.
  • Designing a database where each feature is closely connected to others was challenging.
  • Solution: I actively communicated and collaborated with related divisions to ensure each feature’s data needs were clearly understood and properly integrated into the database structure.
